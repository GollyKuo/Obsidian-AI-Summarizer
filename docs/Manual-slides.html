Preparation

安裝前需要準備的東西

Obsidian

Windows 版 Obsidian 桌面程式。

Vault

一個你已經在使用，或準備用來存放摘要筆記的 Obsidian vault。

Plugin 檔案

AI Summarizer 的 release zip，或已建置好的 plugin 檔案。

不要把這份 HTML 當成 plugin 檔案 Manual-slides.html 只是離線閱讀或投影用的教學簡報，不是 Obsidian plugin 的必要檔案。
API Keys

依照使用情境準備 API key

欄位用途建議
Gemini API Key轉錄與摘要的建議路徑。轉錄模型與摘要模型都建議使用 gemini-2.5-flash。
Mistral API Key摘要備用。Gemini 摘要遇到額度不足、高負載或 503 時使用。
Gladia API Key媒體轉錄備用。Gemini 轉錄遇到高負載、timeout 或不可用時使用，Model 為 default。
最小測試組合 只測網頁摘要：先準備 Gemini。要做媒體摘要：建議先用 Gemini / gemini-2.5-flash，並準備 Mistral 與 Gladia 作為備用路徑。
Release Package

安裝包裡至少應該有三個 plugin 檔案

main.js

外掛主要程式。

manifest.json

Obsidian 用來辨識外掛的資訊檔。

styles.css

外掛介面樣式。

選用檔案 如果安裝包也提供 versions.json，可以一起放進 plugin 資料夾。
Media Workflow

媒體 URL 與本機媒體需要額外工具

如果只使用 網頁 URL 或 文字檔案，通常不需要額外媒體工具。

需要的工具

  • ffmpeg
  • ffprobe
  • yt-dlp
  • 一個放在 vault 外面的媒體暫存資料夾

安裝方式提醒

ffmpeg / ffprobe 與 Windows 桌面版的 yt-dlp 可在 plugin 設定頁自動下載或更新。

也可以自行安裝，使用 自動偵測 或 選擇檔案 指定既有執行檔。

Install Flow

安裝 Plugin 的完整路線

1
解壓縮 release zip

先確認裡面有 main.js、manifest.json、styles.css。

2
找到 Obsidian vault

找到你想安裝外掛的 vault 資料夾。

3
建立 plugin 資料夾

在 vault 裡建立 .obsidian\plugins\ai-summarizer\。

4
複製檔案

把 plugin 檔案放進 ai-summarizer 資料夾。

5
在 Obsidian 啟用

重新開啟 Obsidian，進入 Community plugins 啟用。

Install Step 1

解壓縮安裝包

  1. 下載 AI Summarizer 的 release zip。
  2. 在 Windows 檔案總管中對 zip 檔按右鍵，選擇解壓縮。
  3. 打開解壓縮後的資料夾。
  4. 確認可以看到 main.js、manifest.json、styles.css。
先不要急著複製整包 Obsidian 要讀到的是 plugin 資料夾中的檔案。你要確認最後的路徑裡直接放著 manifest.json，不要多包一層資料夾。
Install Step 2

找到你的 Obsidian vault

Vault 是你的筆記資料夾。它可能放在桌面、文件、雲端同步資料夾，或你自己指定的位置。

  1. 如果你知道 vault 的位置，直接用 Windows 檔案總管打開。
  2. 如果不知道，在 Obsidian 左下角或 vault 管理畫面確認路徑。
  3. Vault 裡應該會有一個隱藏資料夾 .obsidian。
看不到 .obsidian 時 在 Windows 檔案總管開啟 檢視 > 顯示 > 隱藏的項目。
Install Step 3

建立 plugin 資料夾

請在你的 vault 裡建立這個資料夾：

<你的 vault>\.obsidian\plugins\ai-summarizer\

如果 .obsidian\plugins 不存在，可以自己建立。

資料夾名稱要一致 建議使用 ai-summarizer。如果資料夾層級錯誤，Obsidian 會找不到外掛。
Install Step 4

複製 plugin 檔案

把解壓縮後的 plugin 檔案複製到：

<你的 vault>\.obsidian\plugins\ai-summarizer\

最少需要確認這三個檔案都在同一層：

  • main.js
  • manifest.json
  • styles.css
Install Step 5

在 Obsidian 啟用 AI Summarizer

  1. 關閉並重新開啟 Obsidian。
  2. 打開 Settings。
  3. 進入 Community plugins。
  4. 如果 Obsidian 要求關閉 Restricted mode，依畫面提示允許 Community plugins。
  5. 在 Installed plugins 找到 AI Summarizer。
  6. 啟用 AI Summarizer。
啟用後如何打開 可以從左側 ribbon 的 AI 摘要器，或 command palette 的 開啟 AI 摘要器 開啟。
Update

更新 Plugin 時，重點是覆蓋程式檔，不刪設定

一般 Windows 使用者更新時不需要使用終端機。

不要先刪掉整個 ai-summarizer 資料夾 Obsidian 的 plugin 設定檔通常也放在同一個 plugin 資料夾中。如果整個資料夾被刪除，原本的 API key、provider、模型、輸出資料夾與工具路徑設定可能會一起消失。

API key 與使用者設定不在 main.js、manifest.json、styles.css 裡。正常更新時只覆蓋這些 plugin 程式檔，不會清掉 API key。

Update Steps

使用 release zip 更新

  1. 關閉 Obsidian。
  2. 下載新版 AI Summarizer release zip。
  3. 解壓縮 zip。
  4. 打開 <你的 vault>\.obsidian\plugins\ai-summarizer\。
  5. 只用新版 main.js、manifest.json、styles.css 覆蓋同名舊檔案。
  6. 如果新版有提供 versions.json，也一起覆蓋。
  7. 不要刪除其他檔案，特別是設定資料檔。
  8. 重新開啟 Obsidian。
  9. 確認 AI Summarizer 仍然啟用。
  10. 到 Settings > AI Summarizer > AI 模型 確認 API key 還在。
  11. 到 診斷 按 重新檢查。
After Update

更新後建議做一次快速檢查

1
打開 AI 摘要器

確認外掛視窗能正常開啟。

2
跑一次網頁 URL 測試

這是最輕量的基本功能檢查。

3
檢查媒體工具

如果你使用媒體功能，到 診斷 確認 ffmpeg、ffprobe、yt-dlp 都可用。

4
更新模型清單

如果模型 autocomplete 沒有候選，到 AI 模型 分頁按 更新。

Manual Slides

這份 HTML 簡報不會保存設定，也不會自動更新

Settings > AI Summarizer 內的 使用說明 分頁會跟著 plugin 程式一起更新，不需要你額外更新 Manual-slides.html。

如果你是手動下載離線簡報 Manual-slides.html 來教學或投影，想看到新版教學內容時，請另外下載新版簡報檔。

簡單判斷 在 Obsidian 設定頁看說明：更新 plugin 即可。用瀏覽器開 Manual-slides.html：自行下載新版 HTML 才會看到新版簡報。
First Run

第一次設定分成三件事

1. 填 API key

在 AI 模型 分頁填入需要的 API key，並按 測試。

2. 設定輸出位置

決定摘要筆記要寫到 vault 根目錄，或指定資料夾。

3. 設定媒體暫存

如果會處理 YouTube、podcast 或本機媒體，請設定 vault 外的暫存資料夾。

AI Model

填入並測試 API key

  1. 打開 Obsidian Settings。
  2. 進入 AI Summarizer。
  3. 到 AI 模型 分頁。
  4. 填入 Gemini API Key。
  5. 將摘要 Provider 切到 Mistral 時，填入 Mistral API Key。
  6. 將轉錄 Provider 切到 Gladia 時，填入 Gladia API Key。
  7. 分別按 API key 旁的 測試，確認 key 可用。
建議日常媒體設定 轉錄 Provider 建議用 Gemini，轉錄 Model 選 gemini-2.5-flash；Gladia / default 作為媒體轉錄備用。摘要 Provider 建議用 Gemini，摘要 Model 選 gemini-2.5-flash；Mistral 作為摘要備用。
Output

設定摘要筆記輸出位置

  1. 到 輸出與媒體 分頁。
  2. 輸出資料夾 可留空，表示摘要筆記寫到 vault 根目錄。
  3. 也可以按 搜尋資料夾，選擇想放摘要筆記的資料夾。
建議 如果你想讓摘要筆記集中管理，可以建立類似 Summaries 或 AI Notes 的資料夾，再設定為輸出位置。
Media Cache

設定媒體暫存資料夾

如果你要處理 YouTube、podcast 或本機影片音訊，請設定媒體暫存資料夾。

  1. 到 輸出與媒體 分頁。
  2. 找到 媒體暫存資料夾。
  3. 按 選擇資料夾。
  4. 選一個 vault 外面的資料夾。
D:\AI-Summarizer\media-cache
不要放在 vault 裡 避免大量暫存檔進入筆記同步，造成同步變慢或佔用空間。
Diagnostics

第一次使用媒體功能前，先檢查工具

  1. 打開 Settings > AI Summarizer > 診斷。
  2. 找到 媒體工具路徑。
  3. 在 yt-dlp、ffmpeg 或 ffprobe 列按 自動填入。
  4. 等待外掛下載或更新媒體工具。
  5. 若你已自行安裝工具，也可以按 自動偵測 或用 選擇檔案 指到既有執行檔。
  6. 按 媒體功能檢查 的 重新檢查。
  7. 確認 media_url 與 local_media 顯示可用後，再開始媒體摘要。
Diagnostics Notes

媒體工具檢查的三個注意事項

ffmpeg / ffprobe

自動下載主要支援 Windows 桌面版。

yt-dlp

可留空使用系統 PATH，也可用 自動填入 建立 managed install。

media cache root

如果顯示不可用，請重新設定媒體暫存資料夾。

Workflow

使用流程：網頁 URL

  1. 開啟 AI 摘要器。
  2. 選 網頁 URL。
  3. 輸入 http 或 https 網址。
  4. 按 開始摘要。
  5. 完成後按 開啟筆記 檢查結果。
最適合第一次測試 這個流程不需要媒體工具，適合用來確認 API key、摘要 provider 與筆記輸出是否正常。
Workflow

使用流程：媒體 URL

  1. 到設定頁確認轉錄 Provider 與 Model 可用；建議使用 Gemini / gemini-2.5-flash，Gladia / default 作為備用。
  2. 確認摘要 Provider 與 Model 可用；建議使用 Gemini / gemini-2.5-flash，Mistral 作為備用。
  3. 到 診斷 分頁確認媒體工具可用。
  4. 開啟 AI 摘要器。
  5. 選 媒體 URL。
  6. 輸入 YouTube、podcast 或直接媒體網址。
  7. 按 開始摘要。
  8. 完成後按 開啟筆記。
Workflow

使用流程：本機媒體

  1. 到設定頁確認轉錄 Provider 與 Model 可用；建議使用 Gemini / gemini-2.5-flash，Gladia / default 作為備用。
  2. 確認摘要 Provider 與 Model 可用；建議使用 Gemini / gemini-2.5-flash，Mistral 作為備用。
  3. 到 診斷 分頁確認媒體工具可用。
  4. 開啟 AI 摘要器。
  5. 選 本機媒體。
  6. 輸入檔案絕對路徑，或按 選擇檔案。
  7. 按 開始摘要。
媒體摘要的前置條件 本機媒體也需要 ffmpeg、ffprobe、媒體暫存資料夾，以及可用的轉錄 Provider。
Workflow

使用流程：文字檔案

  1. 開啟 AI 摘要器。
  2. 選 文字檔案。
  3. 輸入 .md 或 .txt 文字檔案路徑，或按 選擇檔案。
  4. 摘要建議使用 Gemini / gemini-2.5-flash，Mistral 作為備用。
  5. 按 開始摘要。

這個流程會跳過轉錄，直接用目前摘要 Provider 產生筆記。

適合使用時機 網頁被 403、登入牆、付費牆或動態內容阻擋時，可先複製正文存成 .txt / .md 再摘要；媒體轉錄已完成但摘要失敗、想手動修正逐字稿、或想改用 Mistral 備用路徑重新摘要時，也可以用這個流程。
Settings

設定頁說明：AI 模型

這裡設定轉錄與摘要使用的服務。

  • 轉錄 Provider 建議使用 Gemini，轉錄 Model 選 gemini-2.5-flash。
  • Gladia 是媒體轉錄備用 Provider，Model 使用 default。
  • 摘要 Provider 建議使用 Gemini，摘要 Model 選 gemini-2.5-flash。
  • Mistral 是摘要備用 Provider，適合 Gemini 額度不足或高負載時使用。
  • OpenRouter 是選用，只有切到 OpenRouter 摘要時才需要填 key。
模型清單更新 模型清單更新 的 更新 會重新抓取 Gemini、OpenRouter、Mistral 的官方模型清單，供 autocomplete 使用。

Settings

設定頁說明：使用說明與獨立 HTML 簡報

FAQ

我看不到 .obsidian 資料夾

這通常是因為 Windows 檔案總管預設隱藏了隱藏項目。

1
打開 Windows 檔案總管

進入你的 Obsidian vault 資料夾。

2
開啟隱藏項目

選 檢視 > 顯示 > 隱藏的項目。

3
再檢查一次

你應該會看到 .obsidian 資料夾。

FAQ

Obsidian 裡找不到 AI Summarizer

請依序檢查：

  1. 檔案是否放在 <vault>\.obsidian\plugins\ai-summarizer\。
  2. manifest.json 是否在這個資料夾裡。
  3. Obsidian 是否已重新啟動。
  4. Settings > Community plugins 是否允許 Community plugins。
最常見錯誤 解壓縮後多包了一層資料夾，導致 manifest.json 不在 ai-summarizer 的第一層。
FAQ

Gemini 常常失敗或額度不足

轉錄與摘要都先建議使用 Gemini 的 gemini-2.5-flash。如果 Gemini 遇到額度不足、高負載或 503，再切到備用 provider。

轉錄備用

轉錄 Provider 可改成 Gladia，Model 使用 default。

摘要備用

摘要 Provider 可改成 Mistral，並測試 API key 與模型。

FAQ

媒體 URL 或本機媒體不能用

先到 Settings > AI Summarizer > 診斷：

  1. 按 重新檢查。
  2. 確認 ffmpeg、ffprobe、yt-dlp 可用。
  3. 確認媒體暫存資料夾可用。
  4. 確認轉錄 Provider 與 Model 可用；建議使用 Gemini / gemini-2.5-flash，Gladia / default 作為備用。
  5. 確認摘要 Provider 與 Model 可用；建議使用 Gemini / gemini-2.5-flash，Mistral 作為備用。
FAQ

摘要失敗，但已經有逐字稿

選 文字檔案，指定 transcript.md 或手動整理的文章文字檔，再按 開始摘要。

這樣會跳過轉錄 外掛會只重跑摘要與筆記輸出。這很適合媒體下載或轉錄已經完成，但最後摘要階段失敗的情況。
Errors

常見錯誤代碼

錯誤意思處理方式
validation_error輸入格式不合法。檢查 URL、檔案路徑或副檔名。
runtime_unavailable目前環境或工具不可用。到診斷分頁重新檢查。
download_failure媒體下載或轉檔失敗。檢查網址、yt-dlp 與 ffmpeg。
ai_failureAPI key、provider、模型或額度問題。測試 API key，或換 provider / 模型。
note_write_failure筆記寫入失敗。檢查輸出資料夾是否存在且可寫入。
